# Requirements

> Layer 1 — Specification. The detailed, testable requirements behind
> `project-goal.md`. Each requirement should be verifiable in `verification/`.

## Format

Write requirements so each one can pass or fail a test. Prefer
"The system shall …" or user-story form. Tag with a stable ID so decisions,
tests, and checkpoints can reference it.

| ID | Requirement | Priority | Verified by |
|----|-------------|----------|-------------|
| R1 |             | Must     |             |
| R2 |             | Should   |             |
| R3 |             | Could    |             |

Priority uses MoSCoW: **Must / Should / Could / Won't (this time)**.
