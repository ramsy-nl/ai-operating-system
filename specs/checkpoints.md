# Checkpoints

> Layer 1 — Specification. Work is broken into small, independently reviewable
> checkpoints. Each one is a unit you could ship, review, or roll back on its
> own. Don't start a checkpoint until the previous one has passed verification.

## Rules

- A checkpoint should be completable in one focused session.
- Each checkpoint names its **verification** (how we'll know it's done) and any
  **approval gate** it triggers (`guardrails/ask-first.md`).
- Mark a checkpoint `Done` only after verification passes with real evidence.
